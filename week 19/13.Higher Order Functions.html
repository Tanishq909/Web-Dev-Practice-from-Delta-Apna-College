Higher Order Functions

A function that does one or both :
* takes one or multiple functions as arguments.
* returns a function.

Takes one or multiple functions as arguments.

Example 1: Multiple Greet Function

In the first example, we are dealing with a function named multipleGreet that takes two parameters: a function func and a number n. The purpose of multipleGreet is to execute the function func exactly n times.

Here's a step-by-step breakdown:

Definition of multipleGreet: This function iterates from 1 to n (inclusive) and calls the function func during each iteration.
Definition of greet: This is a simple function that logs "hello" to the console.
Invocation of multipleGreet: We call multipleGreet with the greet function and the number 1234. As a result, "hello" will be printed to the console 1234 times.
This example illustrates how functions can be passed as arguments to other functions and be executed multiple times based on a given condition.

Example 2: Odd or Even Factory

The second example showcases a function called oddOrEvenFactory that returns different functions based on the input request. It demonstrates the concept of a higher-order function, which returns other functions based on specific conditions.

Here's a detailed explanation:

Definition of oddOrEvenFactory: This function takes a single parameter, request. Depending on whether request is "odd" or "even", it returns a function that checks if a given number n is odd or even.
If request is "odd", it returns a function that logs true if n is odd and false otherwise.
If request is "even", it returns a function that logs true if n is even and false otherwise.
If request is neither "odd" nor "even", it logs "wrong request".
Invocation of oddOrEvenFactory: We pass the string "odd" to oddOrEvenFactory. The returned function is not immediately invoked with a number, but we print the function itself to the console.
This example highlights how we can create and return functions dynamically based on input parameters, a powerful feature in JavaScript that enables higher-order programming and functional design patterns.
